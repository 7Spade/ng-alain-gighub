## 🔄 歸檔原則

文檔被歸檔的原因通常包括：

1. **已完成的工作總結** - 記錄特定時期的開發工作，已整合到主要文檔
2. **過期的版本變體** - 被標準版本取代的早期或替代版本
3. **臨時分析報告** - 已完成分析，結論已整合到主要文檔
4. **歷史記錄** - 保留供參考，但不再是當前文檔的一部分

## 📚 如何使用歸檔文檔

### 查找歷史決策
如果需要了解某個功能的實施細節或決策背景：
1. 查看對應日期的工作總結文檔
2. 搜索特定關鍵字（如 "RLS"、"藍圖系統"、"帳戶"）

### 了解演進歷史
如果需要了解系統如何演進到當前狀態：
1. 按日期順序閱讀工作總結
2. 參考實施總結文檔了解各模塊的完整實施過程

### 參考早期設計
如果需要了解早期設計思路：
1. 查看策略與分析報告
2. 參考架構圖的變體版本
