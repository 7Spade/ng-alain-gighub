# 用戶故事目錄

> **目的**：存放項目的用戶故事和功能需求，為 AI 提供業務需求的上下文。

## 📝 用戶故事格式

遵循標準 User Story 格式：

```
As a [角色]
I want [功能]
So that [價值/目標]
```

## 📋 驗收標準

每個用戶故事應包含：

1. **描述**：User Story 描述
2. **驗收標準**：具體的驗收條件
3. **技術要求**：技術實現要求
4. **相關文檔**：相關的架構或設計文檔

## 📁 文件命名規範

- 格式：`story-{feature-name}.md`
- 範例：`story-cert-upload.md`, `story-cert-list.md`
