# 業務上下文

> **目的**：為 AI 提供業務領域知識和上下文，幫助理解項目的業務需求和核心概念。

## 🎯 業務領域

本系統是一個**企業級協作平台**，採用 Git-like 分支模型管理項目和任務。

## 🔑 核心業務概念

### Git-like 分支模型

- **藍圖（Blueprint）**：主項目，類似 Git 的主分支
- **組織分支（Organization Branch）**：協作組織的工作分支
- **PR（Pull Request）**：分支合併請求，需要審核
- **暫存區（Staging Area）**：48 小時可撤回的暫存區域

### 任務管理

- **任務（Task）**：樹狀結構的任務
- **待辦中心**：五種狀態的任務聚合
- **問題（Issue）**：跨分支同步的問題追蹤

### 組織協作

- **組織（Organization）**：主要協作單位
- **團隊（Team）**：組織內的團隊
- **協作關係**：1:1 承攬關係

## 📊 業務流程

### 藍圖建立流程

1. 創建藍圖（主分支）
2. 設置任務結構
3. 邀請協作組織
4. 協作組織 Fork 分支
5. 執行任務並提交到暫存區
6. 創建 PR 請求合併
7. 審核並合併到主分支

### 任務執行流程

1. 查看待辦中心
2. 承攬任務
3. 執行任務
4. 提交到暫存區（48h 可撤回）
5. 創建 PR 或直接合併
